JavaFX Design Pattern Educational Game

Design Patterns Assignment — JavaFX, Delegation, Strategy, Observer Patterns

Project Overview

This educational game was developed as part of the Design Patterns module and focuses on applying object-oriented principles and multiple software design patterns in a JavaFX-based application. The game introduces children to wildlife habitats and the butterfly lifecycle through an interactive zoo environment, animated lifecycle transitions, and a quiz system to reinforce learning.

The goal of the assignment was not to produce game art, but to demonstrate effective use of Delegation, Strategy, and Observer patterns in a modular, extensible architecture where new animals, interactions, and learning scenarios can be added with minimal changes to existing code.

Interactive Habitat Map

Butterfly Lifecycle Simulation

Demonstrates the Delegation pattern as the Egg transitions through lifecycle stages.

Interactive Quiz

Multiple-choice questions test player understanding of ecosystems and life cycles.

Key Design Features

  • Delegation Pattern: Used in the Egg class to transition between lifecycle stages (egg → pupa → butterfly) without rewriting logic.
  • Strategy Pattern: Each lifecycle stage implements different behaviour while sharing the same interface.
  • Observer Pattern: The Zoo controller listens for object interactions and updates UI elements such as knowledge score.
  • Extensible Architecture: New animals, lifecycle models, or quiz content can be added by implementing shared interfaces.
  • Education Focus: Mixes interactive learning, visual cues, and reinforcement through quiz assessment.

Selected Code Examples

Delegation Pattern — Butterfly Lifecycle (Egg.java) 
public class Egg extends GameObject implements ButterflyInterface {
    ButterflyInterface delegate;
    protected int age = 0;

    public Egg(double x, double y, GraphicsContext gc) {
        super(x, y, gc);
        img = new Image(Egg.class.getResource("egg.jpeg").toExternalForm());
        update();
        delegate = this;
    }

    public void grow() {
        age += 1;
        if (age == 100) delegate = new Caterpillar(x, y, gc);
        if (age == 200) delegate = new Pupa(x, y, gc);
        if (age == 300) delegate = new Butterfly(x, y, gc);
        delegate.update();
    }
}
Observer-like Interaction Handling (Zoo.java) 
private void handleClick(MouseEvent event) {
    double x = event.getX();
    double y = event.getY();
    for (GameObject obj : list) {
        if (contains(x, y, obj.getX(), obj.getY(), obj.getImage())) {
            showAlert(obj);
            KnowledgeCount++;
            if (KnowledgeCount >= 7) {
                switchToButterflyApp();
            } else {
                updateKnowledgeText();
            }
            break;
        }
    }
}
Polymorphism via Animal Interface 
public interface AnimalInterface {
    void update();
}

public class Bear extends GameObject implements AnimalInterface {
    public Bear(double x, double y, GraphicsContext gc) {
        super(x, y, gc);
        img = new Image(Bear.class.getResource("bear.png").toExternalForm());
        update();
    }
}
